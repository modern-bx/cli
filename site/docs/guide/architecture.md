# Архитектура и общая логика

## Запуск приложения

Точка входа CLI создаёт контейнер зависимостей и Symfony Console Application. Контейнер регистрирует переводчик, загрузчик команд, сервисы для работы с конфигурацией, базой данных, удалёнными проектами и Bitrix admin API.

## Динамическая загрузка команд

`DynamicCommandLoader` получает карту команд от `CommandFinder`: команда определяется по классу, имени и доступности сервиса в контейнере. Это позволяет собирать PHAR с ограниченным набором команд без изменения пользовательского интерфейса Symfony Console.

После реорганизации команд верхний уровень пространства имён разделён по области ответственности:

- `ModernBx\Cli\App\Console\Command\Core` — базовые команды, не требующие document root Bitrix: completion, JSON/dotenv, remote registry и выбор remote для shell-сессии.
- `ModernBx\Cli\App\Console\Command\Bx` — команды для Bitrix-проекта и связанных операций: подключение ядра, база данных, файлы, сайты, модули, опции, настройки, PHP-код и резервные копии.

## Общий жизненный цикл команды

Базовый `AppCommand` выполняет общие действия:

1. Создаёт `Printer` для единообразного вывода сообщений.
2. Определяет режим подробного вывода через стандартный `--verbose`.
3. Применяет remote текущей shell-сессии, если он задан через переменную окружения `BX_CLI_REMOTE`.
4. Вызывает внутреннюю реализацию конкретной команды.
5. Перехватывает исключения, печатает сообщение об ошибке и возвращает код ошибки.

## Bitrix-aware команды

Команды, наследующие Bitrix-логику, ищут каталог `bitrix` вверх по дереву от текущей директории. Если каталог найден, document root считается родительским каталогом `bitrix`, затем подключается `bitrix/modules/main/include/prolog_before.php`.

Эта логика важна для команд `cache:*`, `module:*`, `option:*`, `setting:*`, `site:*`, `db:*`, `file:*` в локальном режиме и локального `php:exec`. Команда `backup:extract` находится в Bitrix-неймспейсе по предметной области, но распаковывает переданный архив напрямую и не подключает ядро проекта.

## Работа с конфигурацией Bitrix

- `.settings.php` и `.settings_extra.php` читаются как PHP-файлы, возвращающие массив.
- Пути настроек задаются dot-separated синтаксисом без корневого сегмента `value`.
- Значения для записи декодируются из JSON, если это возможно; иначе сохраняются строкой.
- Подключение к БД определяется из `.settings.php`; при необходимости используется старый `php_interface/dbconn.php`.

## Dot-separated JSON path

Команды `json:get`, `json:set`, `setting:get`, `setting:set` используют одинаковую идею пути: сегменты разделяются точкой, а точку внутри имени ключа можно экранировать обратным слешем.

Пример:

```bash
echo '{"a.b":{"c":1}}' | php cli.phar json:get 'a\.b.c'
```

## Удалённое выполнение

Удалённые команды используют зарегистрированные проекты из `~/.config/bx-cli/projects/<codename>/project.yaml`. В конфигурации хранится endpoint, PHPSESSID и срок действия сессии. При истечении сессии менеджер конфигурации пытается обновить её через сохранённые учётные данные.
