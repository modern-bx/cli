# Резервные копии

Команды резервных копий работают с архивами Bitrix и не требуют подключения ядра проекта. Их можно запускать из любого каталога, если пути к архиву и каталогу назначения указаны явно.

## `backup:extract [--password=<password>] <archive> <destination>`

Распаковывает tar-архив резервной копии Bitrix в каталог назначения. Для зашифрованных архивов передайте пароль через `--password`.

```bash
php cli.phar backup:extract ./backup.tar.gz ./restore
php cli.phar backup:extract --password='secret' ./backup.enc.gz ./restore
```

Команда проверяет безопасность путей внутри архива и завершится ошибкой, если запись пытается выйти за пределы каталога назначения. После успешной распаковки результат печатается в JSON-формате.
