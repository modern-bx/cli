# Резервные копии

Команды резервных копий работают с архивами Bitrix и не требуют подключения ядра проекта. Их можно запускать из любого каталога, если пути к архиву и каталогу назначения указаны явно.

## `backup:list [--remote=<codename>] [--local] [--list-all|--list-incomplete]`

Сканирует каталог `/bitrix/backup` внутри найденного Bitrix-проекта и выводит короткие имена основных файлов резервных копий — файлы `.gz` без номера тома, без префикса `/bitrix/backup`. Для дополнительных томов проверяется только последовательность номеров (`.1`, `.2`, `.3`, ...). По умолчанию резервные копии с пропусками томов скрываются. С опцией `--list-all` выводятся все основные файлы, а неполные подсвечиваются красным. С опцией `--list-incomplete` выводятся только неполные резервные копии, тоже красным. Опции `--list-all` и `--list-incomplete` взаимоисключаемы. Целостность архивов и полнота последнего тома не проверяются.

```bash
php cli.phar backup:list
php cli.phar backup:list --list-all
php cli.phar backup:list --list-incomplete
php cli.phar backup:list --remote=prod
```

## `backup:get [--remote=<codename>] [--local] [--force] <backup> <dest>`

Скачивает основной файл резервной копии из `/bitrix/backup` локального или удаленного проекта и все найденные номерные тома (`.1`, `.2`, `.3`, ...) в указанную локальную директорию. Аргумент `backup` — короткое имя основного файла без пути, например `2026-07-20.tar.gz`. Если очередной файл уже существует в директории назначения, команда завершается ошибкой; с `--force` файл перезаписывается. После каждого тома печатается текстовый прогресс с размером скачанного тома и суммарным размером.

```bash
php cli.phar backup:get 2026-07-20.tar.gz ./backups
php cli.phar backup:get --force 2026-07-20.tar.gz ./backups
php cli.phar backup:get --remote=prod 2026-07-20.tar.gz ./backups
```


## `backup:put [--remote=<codename>] [--local] [--force] <src>`

Загружает локальный основной файл резервной копии и все найденные рядом номерные тома (`.1`, `.2`, `.3`, ...) в `/bitrix/backup` локального или удаленного проекта. Аргумент `src` — полный или относительный путь к локальному основному файлу, например `./backups/2026-07-20.tar.gz`. Перед загрузкой команда проверяет последовательность томов; при пропусках завершается ошибкой, кроме режима `--force`. Если файл уже существует в `/bitrix/backup`, команда завершается ошибкой; с `--force` файл перезаписывается. После каждого тома печатается текстовый прогресс с размером загруженного тома и суммарным размером.

```bash
php cli.phar backup:put ./backups/2026-07-20.tar.gz
php cli.phar backup:put --force ./backups/2026-07-20.tar.gz
php cli.phar backup:put --remote=prod ./backups/2026-07-20.tar.gz
```

## `backup:extract [--password=<password>] <archive> <destination>`

Распаковывает tar-архив резервной копии Bitrix в каталог назначения. Для зашифрованных архивов передайте пароль через `--password`.

```bash
php cli.phar backup:extract ./backup.tar.gz ./restore
php cli.phar backup:extract --password='secret' ./backup.enc.gz ./restore
```

Команда проверяет безопасность путей внутри архива и завершится ошибкой, если запись пытается выйти за пределы каталога назначения. После успешной распаковки результат печатается в JSON-формате.
